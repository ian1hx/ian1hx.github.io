關於我

我叫李宜樺，畢業於暨大資工系。畢業後等待入伍半年的時間在銑床廠打工，退伍後，一邊在 MDN 上讀著前端網頁的開發教學，一邊嘗試在英文補習班打工，存下我接下來打算進修的費用。讀資工仍要進修，是因為課程或者專題都較偏理論、數學，缺乏網站開發的實務經驗或作品，為了將各科所學的知識整合，並實際應用，因此才想要進修。而大學的專題，是用 Hamming Code 錯誤更正碼的概念實作資料隱藏，修改少量的元位來藏資料，核心的數學概念是抽象代數。

大學曾在射箭社擔任過社長，舉辦過三場人數約 20 至 50 人的射箭比賽。射箭比賽雖是用手計分，不過正式算總分、排名、驗證分數合理性是以 Excel 的公式處理，大大減少工作人員的工作量，讓比賽過程更為順利。但剛開始不如我所願。在第一場比賽時，因為賽前測試不夠仔細，導致最後計算結果不正確。我得到最大的教訓是：「就算認為絕對不會出錯的地方，也一定要測試，把可能的錯誤降到最低」。

在培訓班進修的時候，剛好偶然得知有兩位在金融業工作的朋友需要自動整理 Excel 資料的 RPA，且公司暫時只能接受 UiPath。UiPath 是以 .NET Framework 為基底的，可以使用 VB 和 C# 來撰寫邏輯，而我當時又正在學 C#，因此我就接下了他們的案子。經過三天的摸索，終於完成委託。UiPath 可以使用視覺化工具兜出程式邏輯，也可以直接撰寫程式碼。經過實測，直接撰寫程式碼的執行速度較快。以迴圈為例，視覺化兜出來的執行 500 次需時 28 秒；直接撰寫程式碼的執行 10000 次儘需 7 秒，近 80 倍的差距。而我將好幾千筆的資料判斷、處理、有迴圈的地方都以程式碼直接撰寫，提升 RPA 的執行效率。

在培訓班所做的專題為「設備租借管理系統」。主要負責資料庫架構和 API 設計；改善密碼儲存機制；前端有使用 Vue.js 撰寫非同步驗證邏輯；串接藍新金流以實現付款功能；串接 LINE Bot 以及使用 Azure Functions 實現定期發送通知給使用者。

我喜歡學習與嘗試新事物，會好奇是否有更好的解決方案。例如在培訓班專題中的「密碼儲存機制」修改了兩次，從原本的密碼加鹽，到使用 PBKDF2，因為好奇是否有更好的密碼儲存機制而花了不少時間讀網路上的英文文件，最後也成功實作出來，讓我相當有成就感。因此我的自我期許是不斷學習，不要放棄可以自我成長的機會。
